快速開始指南

本指南將幫助您快速安裝、配置並開始使用 ATM 自動櫃員機系統。

🖥️ 系統需求

在開始之前，請確保您的系統符合以下最低要求：

1作業系統

  • Windows 10 (64-bit) 或更高版本
  • Windows 11 (推薦)

2硬體需求

  • 處理器：Intel Core i3 或 AMD 同等級處理器
  • 記憶體：至少 2GB 可用 RAM
  • 硬碟空間：至少 500MB 可用空間
  • 螢幕解析度：1024x768 或更高

3開發環境 (僅開發者需要)

  • Qt Creator：6.9.0 或更高版本
  • 編譯器：MinGW 64-bit
  • Qt 框架：Qt 6.9.0 或更高版本

📥 下載與安裝

ATM 系統提供兩種使用方式，請根據您的需求選擇：

重要提醒

一般用戶建議使用「已編譯版本」，無需安裝任何開發工具即可使用。

🚀 方法一：直接執行已編譯版本 (推薦)

適合對象：一般用戶、快速體驗者

優點：無需安裝開發環境，開箱即用

1下載專案

從 GitHub 下載完整專案：

git clone https://github.com/sheng5526cs/ATM_Banking_System.git

或直接下載 ZIP 檔案並解壓縮。

2導航到執行目錄

開啟檔案總管，導航到：

ATM_Banking_System\ATM_Version 1.0\

3執行程式

雙擊 ATM.exe 檔案即可啟動系統。

完成！

如果一切正常，您應該會看到 ATM 登入介面。

🔧 方法二：從原始碼編譯 (開發者)

適合對象：開發者、學習者、需要修改程式碼的用戶

需求：Qt Creator 開發環境

1安裝 Qt Creator

  1. 前往 Qt 官方網站
  2. 下載並安裝 Qt 開源版本
  3. 安裝時確保勾選：
    • Qt 6.9.0 或更高版本
    • MinGW 64-bit 編譯器
    • Qt Creator IDE

2開啟專案

  1. 啟動 Qt Creator
  2. 選擇 File → Open File or Project
  3. 導航到 ATM_Banking_System/ATM_Code/
  4. 選擇 ATM.pro 檔案並開啟

3配置專案

  1. 選擇 Qt 6.9.0 MinGW 64-bit 套件
  2. 確認編譯配置為 Release 或 Debug
  3. 點擊 Configure Project

4建置並執行

  1. 點擊左下角的綠色三角形 ▶️ 按鈕
  2. 等待編譯完成
  3. 程式將自動啟動

編譯注意事項

  • 確保專案路徑不包含中文字元
  • 首次編譯可能需要較長時間
  • 如果編譯失敗，請檢查 Qt 版本和編譯器配置

🎯 首次啟動

當程式成功啟動後，您會看到 ATM 登入介面。

1資料庫初始化

系統會自動檢查並初始化 SQLite 資料庫：

  • 如果是第一次運行，系統會自動建立 atm.db
  • 資料庫會包含預設的測試帳戶
  • 所有必要的資料表都會自動建立

2介面介紹

登入介面包含以下元素：

  • 帳號輸入框：輸入用戶帳號
  • 密碼輸入框：輸入用戶密碼
  • 登入按鈕：驗證並進入系統
  • 清除按鈕：清空輸入框

👤 測試帳戶

系統預設提供以下測試帳戶供您體驗各項功能：

1一般用戶帳戶

帳號: 123
密碼: 3210

功能權限：

  • 登入/登出
  • 餘額查詢
  • 存款作業
  • 提款作業
  • 轉帳功能
  • 交易記錄查詢
  • 密碼變更

2管理員帳戶

帳號: admin
密碼: 5678

功能權限：

  • 所有用戶功能
  • 帳戶管理（新增、停用、啟用）

功能測試建議

建議按以下順序測試功能：

  1. 使用一般用戶帳戶登入
  2. 查詢帳戶餘額
  3. 進行存款操作
  4. 進行提款操作
  5. 嘗試轉帳功能
  6. 查看交易記錄

  8. 體驗管理員專屬功能

🔧 故障排除

如果遇到問題，請參考以下解決方案：

編譯相關問題

問題：編譯時出現錯誤

解決方案：

  • 確保已安裝正確版本的 Qt (6.9.0+) 和 MinGW
  • 檢查專案路徑是否包含中文字元
  • 嘗試清理專案：Build → Clean All
  • 重新建置：Build → Rebuild All

執行相關問題

問題：程式無法啟動或出現 DLL 錯誤

解決方案：

  • 確保使用已編譯版本中的所有檔案
  • 檢查是否安裝了 Visual C++ Redistributable
  • 確認 Qt DLL 檔案在執行檔同一目錄
  • 以系統管理員身分執行

資料庫相關問題

問題：無法連接資料庫或資料遺失

解決方案：

  • 檢查 atm.db 檔案是否存在於執行檔同一目錄
  • 如果資料庫損壞，可以刪除 atm.db 讓程式重新建立
  • 確保對資料庫檔案有讀寫權限

登入相關問題

問題：無法使用測試帳戶登入

解決方案：

  • 確認輸入的帳號密碼正確（區分大小寫）
  • 檢查是否有多次登入失敗導致帳戶鎖定
  • 重新初始化資料庫
